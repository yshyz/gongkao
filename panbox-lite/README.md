# PanBox Lite

本工具是本地运行的网盘工作台，用于处理你有权保存和分享的夸克网盘链接。

## 安全边界

- 默认使用演练模式：只解析链接内容，不转存、不分享。
- 正式任务只会创建一个新的目标文件夹、转存文件、生成分享链接。
- 不包含删除、移动、重命名网盘文件的接口。
- Cookie 只保存在本机 `data/accounts.local.json`，该目录已加入 `.gitignore`。
- 结果导出到本机 `exports/`，该目录也已加入 `.gitignore`。
- 每步默认间隔 3 秒，减少连续请求对账号的压力。

## 启动

在 `panbox-lite` 目录运行：

```powershell
npm start
```

然后打开：

```text
http://127.0.0.1:8787
```

## 推荐流程

1. 用“扫码添加”添加账号，或手动添加 Cookie。
2. 点击“测试账号”确认 Cookie 可用。
3. 粘贴每天收到的分享链接。
4. 先保持“演练模式”运行一次，确认能解析到文件。
5. 取消“演练模式”，勾选正式执行确认。
6. 开始任务，等待转存和分享完成。
7. 下载 CSV 结果，用于网站更新或群发。

## 扫码登录

- 扫码会打开独立的 Chrome/Edge 用户目录，不读取你日常浏览器的登录状态。
- 支持扫码入口：夸克、百度、阿里云盘、迅雷云盘。
- 当前自动“批量转存 + 单独分享”只支持夸克账号。
- 百度、阿里云盘、迅雷云盘目前先支持保存扫码凭据，跨盘保存/上传能力后续接入。

## 账号影响说明

工具不能承诺第三方平台永远不触发风控，但第一版已经做了这些保护：

- 不做删除、移动、重命名。
- 每次正式任务创建独立目录，避免混到原有资料里。
- 限速执行，避免高频请求。
- 演练模式默认开启。
- 任务可以停止。

更稳的做法是每天先用少量链接正式测试，再扩大数量。
